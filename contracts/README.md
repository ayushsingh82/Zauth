# ZAuth Contracts

Hardhat project that compiles, tests, and deploys:

- **`ZAuthVerifier.sol`** — Groth16 verifier auto-generated by snarkjs from
  `circuits/zauth.circom`. `verifyProof(uint256[2], uint256[2][2], uint256[2], uint256[4])`
  returns `true`/`false`. Uses only the standard bn128 precompiles (0x06 / 0x07 / 0x08),
  so it runs on any EVM chain — including HashKey Chain testnet.
- **`ZAuthAttestor.sol`** — thin wrapper that calls the verifier, checks
  `block.timestamp <= expiry`, rejects replays (`consumed[nonce]`), and emits
  `ProofAttested(submitter, nonce, expiry, commitment, powHash)`. Calling it
  produces a real transaction you can show on the explorer.

## Install & test locally

```bash
npm install
npm test
# ✔ accepts a real Groth16 proof and rejects a tampered one
# ✔ rejects a proof where a coordinate has been flipped
```

The test spawns the circom prover via `circuits/scripts/prove.mjs` to produce
a real proof, deploys the verifier to Hardhat's in-memory EVM, and asserts
both accept and reject paths.

## Deploy to HashKey Chain testnet

```bash
cp .env.example .env
# fill HASHKEY_PRIVATE_KEY= (funded via https://faucet.hsk.xyz/faucet)
npm run deploy:hashkey
```

Prints the deployed verifier + attestor addresses. A deploy consumes
~0.001 HSK; 0.1 HSK from the faucet is plenty for multiple redeploys.

## Send a real proof tx to live chain

```bash
ATTESTOR_CONTRACT=0x0c9CfcfDfDA0317659DEB59d3409D059a0DCe5DB \
VERIFIER_CONTRACT=0xC40c974E6D50D201C93265a9D8423e30D0C551aE \
npx hardhat run scripts/attest-live.ts --network hashkey
```

Generates a fresh proof, sanity-checks via view call, then submits
`attestor.attest(...)` as a real tx. Prints the tx hash and explorer URL,
decodes the `ProofAttested` event, and confirms replay protection by calling
`staticCall` again.

## Hardhat config notes

- `solidity: 0.8.20`, `evmVersion: paris` — works on HashKey and any
  post-London chain. Avoids PUSH0 issues on older L2s.
- Network `hashkey` reads `HASHKEY_RPC_URL` (default `https://testnet.hsk.xyz`)
  and `HASHKEY_PRIVATE_KEY` from `.env`. ChainId `133`.
